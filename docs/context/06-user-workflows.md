# 06. Пользовательские сценарии

Документ описывает основные end-to-end сценарии глазами пользователя.

## 1) Загрузка sitemap

1. Пользователь вводит URL sitemap в `SitemapInput`.
2. Нажимает `LOAD`.
3. Frontend запускает `/api/sitemap/load/start`.
4. Далее polling `/api/sitemap/load/progress`.
5. После завершения получает результат через `/api/sitemap/load/result`.
6. URL отображаются в `UrlTable`.

Критерий успеха:

- статус в UI меняется на loaded;
- таблица заполнена URL;
- счетчик найденных URL растет в процессе.

## 2) Анализ таблицы URL

Пользователь:

- ищет URL по подстроке;
- сортирует поля;
- фильтрует по `Sitemap File`;
- переключает размер страницы пагинации.

Критерий успеха:

- таблица и счетчики корректно отражают фильтрованные данные.

## 3) SCAN TITLES

1. Пользователь нажимает `SCAN TITLES`.
2. Backend запускает async сканирование.
3. Frontend polling `/api/scan/progress`.
4. По завершении таблица обновляется с title/ошибками.

Критерий успеха:

- большинство строк имеют `scan_status=done` и заполненный `title`;
- ошибки явно отображаются (`scan_error`).

## 4) TEST URL

1. Пользователь нажимает `TEST URL`.
2. Backend выполняет HEAD/fallback GET Range проверки.
3. Frontend получает прогресс через `/api/scan/test-progress`.
4. В таблице появляется статус проверки URL.

Критерий успеха:

- заполнены `test_status`, `test_http_status`, `test_response_time_ms` там, где это возможно.

## 5) Экспорт результатов

1. Пользователь нажимает `EXPORT`.
2. Frontend вызывает `/api/sitemap/export`.
3. Скачивается Excel-файл с результатами.

Критерий успеха:

- файл `.xlsx` открывается и содержит релевантные колонки.

## 6) Архив и восстановление

1. Пользователь открывает модалку `Архив`.
2. Ищет нужную сессию по URL.
3. Нажимает восстановление.
4. Данные выбранной сессии возвращаются в главную таблицу.

Критерий успеха:

- в таблице отображаются URL и результаты выбранной архивной сессии.
