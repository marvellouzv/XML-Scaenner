# 03. Backend сервисы и бизнес-логика

Документ описывает ответственность основных backend-сервисов в `backend/services`.

## `sitemap_parser.py`

Назначение: загрузка и разбор sitemap-дерева.

Ключевые задачи:

- поддержка `urlset` и `sitemapindex`;
- рекурсивный обход дочерних sitemap;
- поддержка `.xml.gz` (декомпрессия и парсинг XML);
- дедупликация URL;
- привязка `source_sitemap` для каждой записи;
- ограничение по `MAX_SITEMAP_URLS`;
- прогресс-колбэк для realtime счетчика найденных URL.

Поведение при ошибках:

- retries для временных сетевых ошибок;
- явная обработка 403 и информативные ошибки через `SitemapParserError`.

## `title_scanner.py`

Назначение: асинхронное сканирование `<title>` у URL в сессии.

Ключевые задачи:

- конкурентное выполнение через `asyncio.Semaphore`;
- сканирование в несколько раундов;
- retry временных ошибок (timeout, 429, 5xx);
- адаптивная деградация: меньше concurrency, больше delay;
- хранение runtime состояния сканирования для диагностического progress API.

Результаты:

- для каждой записи обновляются `title`, `scan_status`, `scan_error`.

## `url_tester.py`

Назначение: безопасная проверка отклика URL без полной загрузки страницы.

Ключевые задачи:

- сначала `HEAD` запрос;
- fallback на `GET` с `Range: bytes=0-0`, если HEAD недоступен (`405`, `501`);
- измерение времени ответа;
- многорундовый retry с адаптивным троттлингом.

Результаты:

- обновляются `test_status`, `test_http_status`, `test_response_time_ms`, `test_error`.

## `crud.py` как сервисный слой доступа к данным

Хотя файл расположен в корне backend, он выполняет роль сервисного data-access слоя:

- создание/обновление сессий;
- bulk вставка URL;
- обновление результатов title/test;
- формирование прогресса и breakdown ошибок;
- агрегаты для архива;
- reset статусов перед повторным запуском операций.
