# Контекст проекта по разделам

Эта папка предназначена для поэтапного знакомства с проектом. Каждый файл описывает отдельную функциональную область, чтобы можно было изучать контекст частями.

## Как читать

1. Начните с `01-product-overview.md`.
2. Для backend логики откройте `02-backend-api.md` и `03-backend-services.md`.
3. Для модели данных откройте `04-data-model.md`.
4. Для frontend архитектуры и UX-сценариев откройте `05-frontend-architecture.md` и `06-user-workflows.md`.
5. Для запуска и эксплуатации откройте `07-run-and-ops.md`.

## Состав папки

- `01-product-overview.md` - назначение продукта, структура репозитория, основные модули.
- `02-backend-api.md` - карта API endpoint-ов и назначение каждого.
- `03-backend-services.md` - бизнес-сервисы backend и их ответственность.
- `04-data-model.md` - схема хранения данных, ключевые сущности и поля.
- `05-frontend-architecture.md` - архитектура frontend, store, hooks, ключевые компоненты.
- `06-user-workflows.md` - пользовательские сценарии end-to-end.
- `07-run-and-ops.md` - локальный запуск, порты, env-переменные, Docker и диагностика.

## Принцип актуальности

При изменениях API, бизнес-логики, структуры данных, UX-сценариев или запуска, соответствующие файлы в этой папке должны обновляться в той же задаче.
